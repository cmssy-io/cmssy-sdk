# Troubleshooting: symptom → cause

Every row here cost us more than half a day. None of them failed a build.

| Symptom                                                                                                               | Cause                                                                                                                                                                    | Fix                                                                                                                                                                                    |
| --------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------ | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **The editor preview is blank.** The site works.                                                                      | A verified editor request is rewritten to `/cmssy-edit/...` and nothing is mounted there.                                                                                | Add `app/cmssy-edit/[[...path]]/page.tsx` with `createCmssyEditPage`. See [v3→v4](migrations/v3-to-v4.md). In dev the SDK now prints this.                                             |
| **I click the header in the editor and the fields panel is empty.**                                                   | The header is a layout **block**, but it was rendered server-side - the editor can select markup it cannot fill.                                                         | Render the layout blocks through the edit bridge in edit mode: `<CmssyLayoutSlot … editable={EditableLayout} />`. Your middleware must also set the edit header on the rewrite (the preset does). |
| **The editor shows the published page, not the draft I am editing.**                                                  | `fetchLayouts` / the page fetch ran without `previewSecret`.                                                                                                             | Pass `previewSecret: cmssy.draftSecret` in edit mode. `CmssyLayoutSlot` does it for you.                                                                                                   |
| **Google is not indexing my translations.** Search Console says "duplicate, submitted URL not selected as canonical". | `buildCmssyMetadata` got a path with the language prefix **stripped**, so it built the canonical for the default language. Every translation pointed at the English URL. | Pass the path **as routed**: `buildCmssyMetadata(cmssy, path)`. (SDK ≥ 4.1.0)                                                                                                          |
| **`cmssy: missing required configuration: CMSSY_ORG_SLUG…` in the browser console**, and the page is blank.           | Not a config problem. A client component imported a **value** from a module that reads the cmssy config, dragging `process.env` into the browser bundle.                 | Import **types only** (they are erased), or move the value into a module that does not touch the config. Types are free; values are not.                                               |
| **The site renders in English although the workspace is Norwegian.**                                                  | `CmssyServerLayout` / `CmssyServerPage` were rendered without a `locale` **and** without a `config` to look one up.                                                      | Pass `config` (the SDK then asks the workspace) or `locale`. (SDK ≥ 4.5.0 warns in dev.)                                                                                               |
| **My products are not in the sitemap.**                                                                               | Products are model **records**, not pages - `createCmssySitemap` only knows about pages.                                                                                 | Add them through `extra`, which receives the same `baseUrl` and locales the page entries use. (SDK ≥ 4.2.0)                                                                            |
| **The whole site is stuck in edit mode.**                                                                             | `isCmssyEditRequest` is async since 3.0.0 and the `await` is missing - the Promise is truthy.                                                                            | `await isCmssyEditRequest(request, cmssy)`, or use the preset.                                                                                                                         |
| **The editor preview renders in the wrong language.**                                                                 | The middleware resolved the locale into a header, and the edit rewrite dropped it.                                                                                       | `cmssyEditRewrite(request, cmssy, { requestHeaders })` (SDK ≥ 4.3.0), or use the preset.                                                                                               |

## The rule behind all of them

Everything above is a **wiring** mistake, and wiring mistakes do not fail builds:
the site compiles, serves, and looks right. That is why the SDK ships
[`checkCmssyEditMode`](testing.md) - a smoke test for the one path a build can
never check.
